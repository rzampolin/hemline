# Marketing / landing decisions (2026-07-13)

Landing polish + `/about`, built to sell the three differentiators (hem moat,
privacy, breadth) in the first ten seconds of a shared link. Voice per
PRODUCT_SPEC: knowledgeable friend, no "AI-powered" fluff.

## URL & structure

1. **`/about`, not `/how-it-works`.** One editorial page covers the fit
   problem, hem math, color analysis, catalog sourcing, and privacy — "about"
   is the honest umbrella; "how it works" is its h1 and the nav label. Landing
   keeps its compact `#how-it-works` strip and deep-links `/about#hem-math`.
   Anchors (`#hem-math`, `#colors`, `#catalog`, `#privacy`) are the contract;
   footer and landing link them — don't rename casually.

2. **Shared `MarketingFooter`** (`apps/web/app/(marketing)/footer.tsx`) on
   landing + about only. The `(app)` shell keeps its bottom tab nav untouched —
   a marketing footer under an infinite-scroll feed is clutter, and a parallel
   workstream is adding a `/check` nav entry; keeping marketing chrome in the
   `(marketing)` group avoids structural conflicts. If `/check` ships, add it
   to `footer.tsx` — one line.

3. **Height-toggle proof strip kept as-is** (it demonstrates the moat better
   than copy can); hero subline rewritten to lead with the hem hook ("Length
   labels describe a fit model, not you").

## Claims policy (the load-bearing part)

4. **Every number and promise on landing + about was verified against code or
   docs before writing** (verification table in the workstream report; key
   ones inline here):
   - *12,800+ dresses / 35 boutiques & brands / hem line on ~92%* — live
     catalog stats supplied by EM 2026-07-13. **These are hardcoded copy** on
     the landing stats band, `/about`, and the OG image — update all three
     together (grep `12,800`).
   - *Refreshed daily* — Shopify/JSON-LD connectors default `0 6 * * *`, eBay
     `0 */6 * * *` (`packages/connectors/src/*/index.ts`), scheduler in-container
     (`docker/entry-scheduler.ts`).
   - *Possibly-sold detection* — `apps/ingest/src/freshness.ts` (unseen 2×
     cadence → soft delete) + "Possibly sold" flags on My Rack.
   - *Selfie discarded* — `apps/web/app/api/color-analysis/route.ts` processes
     in memory only; only season + palette persist.
   - *No third-party trackers* — first-party whitelist-only analytics
     (docs/decisions-analytics.md); no external scripts in `app/layout.tsx`
     (next/font self-hosts).
   - *Affiliate disclosure* — eBay EPN campaign header exists
     (`packages/connectors/src/ebay/index.ts`), so `/about#catalog` discloses
     "some links are affiliate links".

5. **Deliberately NOT claimed:** "delete/reset everything anytime". In live
   mode the httpOnly session cookie survives `reset()`
   (`lib/profile-store.tsx` clears localStorage then `GET /api/session`, which
   returns the *same* users row — server-side quiz answers/swipes reappear),
   so "Reset my profile" is effectively device-cache-only today. The privacy
   section therefore promises visibility/editability (true: profile PATCH)
   instead of deletion. **Flagged to product/backend as a bug**; when a real
   server-side reset ships, strengthen `/about#privacy`.

6. **"Midi spans ~8 inches" grounded in our own band math** (`lib/hem.ts`
   priors: knee 39″ … mid_calf 47″): a 47″ garment is ankle on 5′2″ (r=0.062),
   a 39″ garment is knee on 5′10″ (r=0.263). The about-page prose uses those
   exact figures.

## Meta / assets

7. **`metadataBase` = `NEXT_PUBLIC_SITE_URL` ?? `https://hemline.fly.dev`**;
   root layout sets a title template (`%s · Hemline`), default OG image, and
   `summary_large_image` twitter card. Landing inherits the default title
   (setting a string there would double the brand via the template).

8. **OG image is a committed static** (`apps/web/public/og.png`, 1200×630),
   generated by `apps/web/scripts/generate-og.mjs` (sharp rasterizing an
   in-theme SVG; Georgia stands in for Fraunces since rasterization uses
   system fonts). Regenerate + recommit when copy/stats change; CI never runs
   the script. Favicon is `app/icon.svg` (Next file convention).
